Core Infrastructure Capabilities

Built from the ground up for security, scale, and zero vendor lock-in.

Container Isolation

Every agent gets its own sandboxed Docker container with CPU/memory limits, read-only mounts, and no root access. Cross-agent interference is architecturally impossible.

Multi-Provider AI

Anthropic and OpenAI out of the box. Switch providers without changing agent logic. Any OpenAI-compatible endpoint works via the custom provider.

Swarm Orchestration

Break complex tasks into sub-agent DAGs. The coordinator delegates, aggregates results, and handles failures — all within isolated containers.

Scoped Memory System

Agents maintain context within defined boundaries. Persistent memory at three levels: private (per-user), group (team), and org-wide.

Frequently Asked Questions

Everything you need to know about Clawix.

What is Clawix?

Clawix is a self-hosted multi-agent AI orchestration platform. It allows you to run AI agent swarms in isolated Docker containers, providing full governance, security, and zero vendor lock-in for enterprise deployments.

Is Clawix open source?

Yes, the core Clawix platform is Open Source and released under the MIT License. AIBYML.SG provides premium consulting, enterprise integration, and compliance services on top of the open-core.

Which AI providers are supported?

Clawix supports Anthropic and OpenAI out of the box. Additionally, any OpenAI-compatible endpoint (like Ollama or vLLM) works natively, allowing you to run local LLMs easily. Azure, DeepSeek, and Gemini integrations are on the immediate roadmap.

Does Clawix require Docker?

Yes. Docker is fundamental to Clawix's security model. Every agent runs in its own sandboxed container with CPU/memory limits, ensuring no cross-agent contamination and preventing agents from accessing the host system.
